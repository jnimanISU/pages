The Shared Responsibility Model

This is the most important concept in cloud security. Security is a partnership. The provider is responsible for securing the cloud itself, while you are responsible for securing what you put *in* the cloud.

Provider Responsibility

  • Physical Data Centers
  • Hardware & Infrastructure
  • Virtualization Layer
  • Network Foundation

Customer Responsibility

  • Data Encryption
  • Identity & Access (IAM)
  • Application Security
  • Firewall Configuration
  • Operating System (in IaaS)

The New Mindset: From Perimeter to Zero Trust

The complexity of the cloud requires a shift in our security philosophy. The old "castle and moat" model is obsolete. We must move to a **Zero Trust** model, which assumes no user or device is trusted by default.

Old Model: Perimeter Security

üè∞

"Trust but verify"

Relies on a strong outer firewall. Once inside, users and devices are often trusted, allowing threats to move laterally.

New Model: Zero Trust

üîê

"Never trust, always verify"

Authenticates and authorizes every single request. Access is granular, time-bound, and strictly enforced, minimizing breach impact.
